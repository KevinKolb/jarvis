Tech Jargon

Network

Wi-Fi ...... SpamNet (all devices must be on it)
Raspberry Pi ...... 192.168.86.147 (the web server)
Hue Bridge ...... 192.168.86.151 (kitchen = group 82)
Site ...... http://192.168.86.147/
Kitchen ...... http://192.168.86.147/kitchen/

How it works

The Pi runs server.py on port 80: it serves this website and relays light commands.
Tap a light → phone POSTs to /hue on the Pi → Pi sends a PUT to the Hue bridge. (Browsers can't call the bridge directly.)
Music buttons → iOS Shortcuts (Alexa has no local API).
Code lives at github.com/KevinKolb/jarvis. The Hue token is only in server.py on the Pi, not in the public code.

Update the Pi (from any computer on SpamNet)

ssh USERNAME@192.168.86.147
cd jarvis
git pull
bash setup-pi.sh

Handy commands (run on the Pi)

sudo systemctl restart jarvis   # restart the server
sudo systemctl status jarvis    # is it running?
journalctl -u jarvis -f         # live log (Ctrl+C to quit)
hostname -I                     # the Pi's IP address

If the lights stop working

1. Is the phone on SpamNet?
2. Is the Pi powered on? (it must stay on)
3. Did the Pi's address change after a reboot? If the site won't load, run hostname -I on the Pi to get the new address.
